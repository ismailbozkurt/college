\subsection{Trabajo relacionado}
\label{sec:related_work}
Antes de continuar, cabe mencionar que este documento se cimienta en los conceptos explicados en \textit{Introducción a la explotación de software en sistemas Linux}\cite{IALEDSESL}, escrita como preámbulo de esta investigación.\bigskip

El documento mencionado explica dos conceptos muy diferentes.\\
El primero de ellos es el \textit{shellcoding}. El desarrollo de \textit{shellcodes} se basa en la programación en ensamblador de ciertas rutinas que permitan realizar las acciones que un programador necesite una vez se haya vulnerado el software investigado. La programación de \textit{shellcodes} es bastante compleja ya que cada una de las rutinas programadas debe cumplir ciertas restricciones y, debido a estas restricciones, el programador no puede utilizar todas las funcionalidades que proporciona el lenguaje ensamblador.\\
El segundo concepto tratado explica el modo de vulnerar software aprovechando los desbordamientos de búfers en la región de memoria llamada \textit{stack}. Este tipo de vulneración de software es parecida a la que se explica en esta investigación, pero que implica el \textit{heap}, sin embargo, la explotación de desbordamientos en el \textit{heap} es mucho más ardua.\bigskip

Se recomienda al lector tener un dominio básico de los conceptos tratados en la investigación mencionada para poder seguir la investigación actual sin ningún tipo de problema.
